  <action>
Update settings.py for production:

```python
import os

DEBUG = os.environ.get('DEBUG', 'False') == 'True'

# SECURITY
SECRET_KEY = os.environ.get('SECRET_KEY', 'change-me-in-production')
ALLOWED_HOSTS = os.environ.get('ALLOWED_HOSTS', '').split(',')

# In production, use proper static file serving
if not DEBUG:
    # Use WhiteNoise for static files
    MIDDLEWARE = [
        # ... existing middleware
        'whitenoise.middleware.WhiteNoiseMiddleware',
    ]
    
    STATICFILES_STORAGE = 'whitenoise.storage.CompressedManifestStaticFilesStorage'

# HTTPS/HSTS (only in production)
if not DEBUG:
    SECURE_SSL_REDIRECT = True
    SECURE_HSTS_SECONDS = 31536000
    SECURE_HSTS_INCLUDE_SUBDOMAINS = True
    SESSION_COOKIE_SECURE = True
    CSRF_COOKIE_SECURE = True
```

Install WhiteNoise:
```bash
pip install whitenoise[gunicorn]
```

Update requirements.txt with production dependencies:
```

djangorestframework>=3.14
django-cors-headers>=4.0
psycopg2-binary>=2.9

```

  <name>Task 2: Deploy Django backend to Railway</name>

  <action>
Deploy to Railway:

1. Push backend code to GitHub:
```bash
# Create backend-only directory structure or use monorepo
git add backend/
git commit -m "Add Django backend"
git push origin main
```

2. In Railway dashboard:
   - Create new project
   - Connect to GitHub repository
   - Add PostgreSQL plugin
   - Set environment variables:
     - `DEBUG`: False
     - `SECRET_KEY`: (generated secure key)
     - `ALLOWED_HOSTS`: your-app.railway.app
   - Deploy

3. Verify deployment:
```bash
curl https://your-app.railway.app/api/projects/
```

4. Create admin superuser on Railway:
```bash
railway run python manage.py createsuperuser
```

Or use Django shell:
```bash
railway run python manage.py shell
# Then create user programmatically
```
  </action>

    <automated>curl -s https://YOUR-RAILWAY-APP.railway.app/api/projects/ | head -50</automated>
